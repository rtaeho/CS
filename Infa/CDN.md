콘텐츠를 전 세계 여러 서버에 분산 저장해두고, 사용자와 가장 가까운 서버에서 제공하는 시스템입니다.

## 왜 필요한가?

- 서버가 미국에 있는데 한국에서 접속하면 느림
- 모든 요청이 원본 서버로 몰리면 과부하
- CDN은 사용자 근처에서 콘텐츠를 전달해 속도 향상 + 서버 부하 분산

## 동작 방식

```
사용자 (한국)
    ↓ 요청
CDN 엣지 서버 (서울)
    ├─ 캐시 있음 → 바로 반환 (빠름)
    └─ 캐시 없음 → 원본 서버에서 가져옴 → 캐싱 후 반환
```

## 주로 캐싱하는 콘텐츠

- 이미지, 동영상
- CSS, JavaScript 파일
- 폰트, 정적 HTML
- 다운로드 파일

## 장점

- **속도 향상**: 물리적으로 가까운 서버에서 응답
- **서버 부하 감소**: 원본 서버 요청이 줄어듦
- **가용성**: 한 서버가 죽어도 다른 서버가 대체
- **DDoS 방어**: 트래픽이 분산되어 공격에 강함

## 대표적인 CDN 서비스

- Cloudflare
- AWS CloudFront
- Akamai
- Fastly

## 예시

Netflix는 전 세계에 CDN 서버를 두고 있어서, 한국에서 영화를 스트리밍할 때 미국 본사 서버가 아닌 한국 근처 서버에서 영상을 받아옵니다.
