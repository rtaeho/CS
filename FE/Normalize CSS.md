브라우저마다 다르게 적용되는 기본 스타일 차이를 보정하되, 유용한 기본 스타일은 유지하는 CSS 초기화 방식입니다.

## 핵심 특징

- **차이 보정**: 브라우저 간 기본 스타일 차이를 줄임
- **기본값 유지**: 모든 스타일을 제거하지 않고 합리적인 기본 스타일을 보존
- **낮은 도입 부담**: 처음부터 모든 요소 스타일을 다시 만들 필요가 적음

## 예시

```css
h1 {
  font-size: 2em;
  margin: 0.67em 0;
}

a {
  background-color: transparent;
}
```

## 언제 쓰나

- 빠르게 개발해야 하는 프로젝트
- 브라우저 기본 스타일을 어느 정도 활용하고 싶은 경우
- 콘텐츠 중심 페이지처럼 기본 타이포그래피 흐름이 필요한 경우

## 주의사항

- Reset CSS보다 통제력은 약할 수 있음
- 프로젝트 디자인 시스템과 맞지 않는 기본 스타일은 별도로 덮어써야 함
- Normalize 적용 후에도 브라우저별 세부 차이를 완전히 없애지는 못함

## 핵심 정리

Normalize CSS는 기본 스타일을 전부 제거하지 않고 브라우저 간 차이만 줄이는 방식입니다.
