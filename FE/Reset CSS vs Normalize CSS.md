브라우저 기본 스타일 차이를 줄이기 위한 두 CSS 초기화 방식의 목적과 적용 강도를 비교하는 개념입니다.

## 핵심 특징

- [[Reset CSS]]는 기본 스타일을 강하게 제거해 완전히 통제된 출발점을 만듦
- [[Normalize CSS]]는 브라우저 차이만 보정하고 유용한 기본 스타일은 유지
- 둘 다 브라우저 간 스타일 차이를 줄여 일관된 UI를 만들기 위한 도구

## 비교

| 항목 | Reset CSS | Normalize CSS |
|---|---|---|
| 접근 방식 | 기본 스타일 제거 | 기본 스타일 보정 |
| 통제력 | 높음 | 중간 |
| 초기 작업량 | 큼 | 작음 |
| 기본 접근성/가독성 | 직접 재정의 필요 | 상대적으로 유지 |
| 적합한 경우 | 디자인 시스템, 강한 UI 통제 | 빠른 개발, 콘텐츠 중심 페이지 |

## 선택 기준

- 모든 컴포넌트 스타일을 직접 관리한다면 Reset CSS
- 브라우저 기본 스타일을 일부 활용하고 싶다면 Normalize CSS
- 팀의 디자인 시스템, 접근성 기준, 개발 속도에 따라 선택

## 핵심 정리

Reset CSS는 지우고 다시 만드는 방식이고,
Normalize CSS는 차이만 고치는 방식입니다.

완전한 통제가 필요하면 Reset,
자연스러운 기본값과 빠른 시작이 필요하면 Normalize가 어울립니다.
