인터넷에서 데이터를 암호화해 안전하게 전송하기 위한 보안 프로토콜입니다. 현재는 TLS로 대체되었지만 여전히 SSL이라는 용어가 많이 쓰입니다.

## SSL vs TLS

- **SSL**: 원래 Netscape가 만든 프로토콜 (SSL 3.0까지)
- **TLS**: SSL을 개선한 후속 버전 (TLS 1.0 ~ 1.3)
- 현재 SSL은 보안 취약점으로 사용 중단됨
- "SSL 인증서"라고 부르지만 실제로는 TLS 사용

## 버전 역사

|버전|상태|
|---|---|
|SSL 2.0, 3.0|폐기 (보안 취약)|
|TLS 1.0, 1.1|지원 중단|
|TLS 1.2|현재 많이 사용|
|TLS 1.3|최신, 권장|

## SSL/TLS가 제공하는 보안

|기능|설명|
|---|---|
|**암호화**|데이터를 읽을 수 없게 변환|
|**인증**|서버가 진짜인지 확인|
|**무결성**|데이터가 변조되지 않았음을 보장|

## 동작 원리 (간략)

```
1. 핸드셰이크
   - 암호화 방식 협상
   - 서버 인증서 검증
   - 세션 키 교환

2. 데이터 전송
   - 세션 키로 대칭 암호화
   - 암호화된 데이터 주고받음
```

## 사용되는 곳

- HTTPS (웹)
- SMTPS, IMAPS (이메일)
- FTPS (파일 전송)
- VPN

## SSL 인증서 종류

|종류|검증 수준|용도|
|---|---|---|
|**DV**|도메인만 확인|개인, 소규모|
|**OV**|조직 확인|기업|
|**EV**|엄격한 검증|금융, 대기업|
