암호화와 복호화에 서로 다른 키(공개키, 개인키)를 사용하는 비대칭 암호화 방식입니다.

## 핵심 구조

```
[공개키 - Public Key]  → 누구에게나 공개
[개인키 - Private Key] → 본인만 보관

암호화: 평문 + 공개키 → 암호문
복호화: 암호문 + 개인키 → 평문
```

## 주요 특징

|항목|내용|
|---|---|
|**키 개수**|2개 (공개키 + 개인키)|
|**키 배송 문제**|없음 (공개키는 공개해도 무방)|
|**속도**|느림 (대칭키 대비)|
|**대표 알고리즘**|RSA, ECC|

## 활용 1 - 암호화 통신

```
A가 B의 공개키로 암호화 → B만 개인키로 복호화 가능
→ 제3자가 가로채도 복호화 불가
```

## 활용 2 - 전자서명

```
서명: 개인키로 서명 생성
검증: 공개키로 서명 검증 → 송신자 인증
```

## 실사용 예시

|사례|내용|
|---|---|
|**HTTPS**|대칭키를 공개키로 안전하게 교환|
|**JWT**|RS256 알고리즘에서 개인키 서명, 공개키 검증|
|**SSH**|서버 접속 시 공개키 인증|

> 속도가 느리기 때문에 실제 데이터 암호화는 **대칭키**로 하고, 그 대칭키를 교환할 때만 공개키를 활용하는 방식을 주로 사용합니다.
