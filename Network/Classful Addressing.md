IP 주소를 **고정된 클래스(A, B, C, D, E)로 나누어 네트워크와 호스트를 구분**하는 초기 IP 주소 체계입니다.

## 클래스 구분

```
IP 주소 첫 번째 옥텟(8bit)으로 클래스 구분

A클래스: 0xxx xxxx → 0~127
B클래스: 10xx xxxx → 128~191
C클래스: 110x xxxx → 192~223
D클래스: 1110 xxxx → 224~239 (멀티캐스트)
E클래스: 1111 xxxx → 240~255 (예약)
```

## A, B, C 클래스 상세

|클래스|범위|네트워크|호스트|호스트 수|
|---|---|---|---|---|
|**A**|0~127|8bit|24bit|약 1600만|
|**B**|128~191|16bit|16bit|약 65000|
|**C**|192~223|24bit|8bit|254|
-> 가장 첫 옥탯으로 클래스를 구분
## 구조 시각화

```
A클래스: [네트워크(8)] [      호스트(24)      ]
B클래스: [  네트워크(16)  ] [    호스트(16)    ]
C클래스: [      네트워크(24)      ] [호스트(8)]
```
-> 각 클래스별로 네트워크 비트 수가 고정
-> 이로 인한 낭비가 심함
## 예시

```
A클래스: 10.0.0.1
         10       → 네트워크 주소
         0.0.1    → 호스트 주소

B클래스: 172.16.0.1
         172.16   → 네트워크 주소
         0.1      → 호스트 주소

C클래스: 192.168.0.1
         192.168.0 → 네트워크 주소
         1         → 호스트 주소
```

## 문제점

```
A클래스: 호스트 약 1600만개
→ 실제로 1000개만 필요한 기관에 할당
→ 나머지 1599만개 낭비

C클래스: 호스트 254개
→ 1000개 필요한 기관엔 부족
→ C클래스 4개 할당해야 함

→ IP 주소 낭비 심각
→ 이를 해결하기 위해 CIDR(클래스리스) 등장
```

## Classful vs Classless (CIDR)

||Classful|CIDR|
|---|---|---|
|**구분 방식**|고정 클래스|서브넷 마스크로 유동적 구분|
|**IP 낭비**|심각|최소화|
|**유연성**|낮음|높음|
|**표기**|`192.168.0.0`|`192.168.0.0/24`|

> Classful 주소 체계는 IP 낭비 문제로 현재는 사용하지 않으며, **[[CIDR]](Classless Inter-Domain Routing)** 방식으로 대체되었습니다.
