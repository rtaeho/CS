서브넷 마스크를 **슬래시(/)와 숫자로 간결하게 표현**하는 표기법입니다.

## 표기 방식

```
192.168.0.0/24
────────────  ──
  IP 주소    네트워크 비트 수

/24 = 앞에서부터 24bit가 네트워크 주소
```

## 서브넷 마스크와 비교

```
전통적 표기: 192.168.0.0  255.255.255.0
CIDR 표기:   192.168.0.0/24
→ 둘 다 같은 의미, CIDR이 더 간결
```

## 비트로 이해하기

```
/24
11111111.11111111.11111111.00000000
← 1이 24개 → ← 0이 8개 →
= 255.255.255.0

/22
11111111.11111111.11111100.00000000
← 1이 22개 →  ← 0이 10개 →
= 255.255.252.0
```

## 호스트 수 계산

```
호스트 수 = 2^(32 - 프리픽스) - 2
-> 네트워크 주소와 브로드캐스트 주소는 실제 장치에 할당할 수 없기 때문에 2를 빼줌

/24 → 2^8  - 2 = 254개
/22 → 2^10 - 2 = 1022개
/30 → 2^2  - 2 = 2개 (라우터 간 연결)
/32 → 2^0  - 2 = 0개 (단일 호스트 지정)
```

## 자주 쓰이는 CIDR

|표기|서브넷 마스크|호스트 수|용도|
|---|---|---|---|
|`/8`|255.0.0.0|약 1600만|대규모 ISP|
|`/16`|255.255.0.0|65534|대기업|
|`/24`|255.255.255.0|254|일반 네트워크|
|`/30`|255.255.255.252|2|라우터 간 연결|
|`/32`|255.255.255.255|1|특정 호스트 지정|

## 네트워크 주소 계산

```
IP와 서브넷 마스크를 AND 연산

IP:     192.168.1.130 → 11000000.10101000.00000001.10000010
/26:    255.255.255.192 → 11111111.11111111.11111111.11000000
AND:    192.168.1.128   → 11000000.10101000.00000001.10000000
                                                      ↑↑
                                               네트워크 비트
```

> CIDR 표기법은 서브넷 마스크를 간결하게 표현하는 방법으로, 클래스리스 주소 체계에서 **네트워크 크기를 명시적으로 나타내기 위해** 사용합니다.
