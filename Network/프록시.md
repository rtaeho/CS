클라이언트와 서버 사이에서 요청과 응답을 대신 중계해주는 중간 서버입니다.

## 핵심 개념

클라이언트가 직접 서버에 접근하지 않고, 프록시 서버를 거쳐 통신함으로써 **보안, [[캐싱]], [[로드밸런싱]]** 등의 이점을 얻습니다.

## [[포워드 프록시]] vs [[리버스 프록시]]

| 항목    | 포워드 프록시          | 리버스 프록시           |
| ----- | ---------------- | ----------------- |
| 위치    | 클라이언트 앞          | 서버 앞              |
| 주체    | 클라이언트가 사용        | 서버 측에서 운영         |
| 목적    | IP 우회, 접근 제어, 캐싱 | 로드밸런싱, SSL 처리, 캐싱 |
| 서버 인지 | 서버는 실제 클라이언트를 모름 | 클라이언트는 실제 서버를 모름  |
| 예시    | 사내 방화벽, VPN      | Nginx, AWS ALB    |
|       |                  |                   |

## 흐름도

```
[포워드 프록시]
클라이언트 → 프록시 서버 → 인터넷 → 목적지 서버

[리버스 프록시]
클라이언트 → 인터넷 → 프록시 서버(Nginx 등) → 내부 서버들
```

## 주요 활용

- **캐싱**: 동일 요청에 대해 프록시가 저장된 응답을 반환하여 속도 향상
- **로드밸런싱**: 리버스 프록시가 여러 서버에 트래픽을 분산
- **보안**: 실제 서버/클라이언트의 IP를 숨기고, SSL 종료 처리
- **접근 제어**: 특정 사이트 차단 또는 허용 목록 관리
