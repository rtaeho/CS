공인 IP 풀(Pool)에서 사설 IP에 **동적으로 공인 IP를 할당**하는 NAT 방식입니다.

## 동작 방식

```
공인 IP 풀: [203.0.113.1, 203.0.113.2, 203.0.113.3]

192.168.0.1 접속 → 203.0.113.1 할당
192.168.0.2 접속 → 203.0.113.2 할당
192.168.0.3 접속 → 203.0.113.3 할당

192.168.0.1 접속 종료 → 203.0.113.1 반환 (풀로 돌아감)
192.168.0.4 접속 → 203.0.113.1 재할당
```

## Static vs Dynamic vs PAT

||Static NAT|Dynamic NAT|PAT|
|---|---|---|---|
|**매핑**|1:1 고정|1:1 동적|다:1|
|**공인 IP**|장치 수만큼|풀(Pool)|1개|
|**동시 접속**|항상 가능|풀 소진 시 불가|가능|
|**외부 접근**|O|X|X|

## 단점

```
공인 IP 풀이 소진되면
→ 새로운 장치는 인터넷 접속 불가

장치 10개, 공인 IP 3개
→ 동시에 3개만 인터넷 접속 가능
→ 나머지 7개는 대기
```

## Static NAT와의 차이

```
Static:  192.168.0.1 → 항상 203.0.113.1 (고정)
Dynamic: 192.168.0.1 → 접속할 때마다 다른 공인 IP 할당
                     → 외부에서 주소 예측 불가
```

> Dynamic NAT는 Static보다 공인 IP를 효율적으로 사용하지만, 풀 소진 문제와 외부 접근 불가 문제가 있어 현재는 **PAT(NAPT)** 방식이 훨씬 많이 사용됩니다.
