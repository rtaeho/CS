사람이 읽을 수 있는 도메인 이름(www.google.com)을 컴퓨터가 이해하는 IP 주소(142.250.196.4)로 변환해주는 시스템입니다. 흔히 "인터넷의 전화번호부"라고 비유합니다.

## 왜 필요한가?

- 사람은 숫자로 된 IP 주소를 기억하기 어려움
- 컴퓨터는 도메인 이름으로 직접 통신할 수 없음
- DNS가 이 둘 사이를 연결해주는 역할

## DNS 조회 과정

```
사용자가 www.google.com 입력
           ↓
    1. 로컬 캐시 확인
       (브라우저 → OS → 라우터)
           ↓ (없으면)
    2. Local DNS 서버에 질의
       (보통 ISP가 제공)
           ↓ (없으면)
    3. Root DNS 서버
       → ".com은 여기서 찾아봐"
           ↓
    4. TLD DNS 서버 (.com 담당)
       → "google.com은 여기서 찾아봐"
           ↓
    5. 권한 있는 DNS 서버 (Google 소유)
       → "IP는 142.250.196.4야"
           ↓
    6. 결과를 캐싱하고 반환
```

## DNS 레코드 종류

|레코드|설명|
|---|---|
|**A**|도메인 → IPv4 주소|
|**AAAA**|도메인 → IPv6 주소|
|**CNAME**|도메인 → 다른 도메인 (별칭)|
|**MX**|메일 서버 지정|
|**NS**|네임서버 지정|
|**TXT**|텍스트 정보 (인증 등에 사용)|

## 캐싱과 TTL

- DNS 결과는 **[[TTL]](Time To Live)** 동안 [[캐싱]]됨
- 매번 조회하지 않아 속도가 빨라짐
- TTL이 지나면 다시 조회

## 알아두면 좋은 점

- DNS는 **[[UDP]] 포트 53번** 사용 (기본)
- 응답이 크면 [[TCP]]로 전환
- 공용 DNS: Google(8.8.8.8), Cloudflare(1.1.1.1)
