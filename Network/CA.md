디지털 인증서를 발급하고 서명하여 서버(혹은 개인)의 신원을 보증해주는 신뢰할 수 있는 제3자 기관입니다.

## 역할

```
서버: "나는 google.com 이고, 이게 내 공개키야"
CA:   "내가 검증했고 보증할게" → 인증서 발급 (CA 개인키로 서명)
클라이언트: "CA가 보증했으니 믿을 수 있다" O
```

## 신뢰 체계 (Chain of Trust)

```
Root CA
├── 브라우저/OS에 공개키 내장
└── Intermediate CA 서명
        └── 서버 인증서 서명
```

> Root CA는 전 세계적으로 신뢰받는 기관으로, 브라우저와 OS에 **미리 내장**되어 있습니다.

## 대표적인 CA 기관

|기관|특징|
|---|---|
|**DigiCert**|가장 널리 사용되는 상업 CA|
|**Let's Encrypt**|무료 인증서 발급, 자동화 지원|
|**Comodo**|저렴한 가격의 상업 CA|
|**GlobalSign**|기업용 CA|

## 인증서 종류

|종류|검증 수준|특징|
|---|---|---|
|**DV (Domain Validation)**|낮음|도메인 소유 확인만|
|**OV (Organization Validation)**|중간|기업 정보 추가 확인|
|**EV (Extended Validation)**|높음|엄격한 심사, 브라우저 주소창 녹색 표시|

## CA가 없다면?

```
누구나 "나는 google.com이야" 라고 주장 가능
→ 중간자 공격(MITM) 방어 불가
```

## 인증서 검증 과정

```
1. 서버로부터 인증서 수신
2. 인증서의 CA 서명을 CA 공개키로 검증
3. 인증서 유효기간 확인
4. 인증서 폐지 여부 확인 (CRL / OCSP)
5. 모두 통과 → 신뢰 O
```

> CA를 신뢰하는 근거는 **브라우저/OS 제조사(Google, Apple, Microsoft 등)가 엄격한 심사를 거쳐 Root CA 목록을 관리**하기 때문입니다.
