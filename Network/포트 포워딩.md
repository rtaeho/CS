외부에서 들어오는 특정 포트의 요청을 **내부 사설 IP의 특정 포트로 전달**하도록 공유기에 설정하는 기능입니다.

## 왜 필요한가?

```
NAT 환경에서 외부 → 내부 접근 기본적으로 불가
→ 공유기가 어느 장치로 전달할지 모름

포트 포워딩으로 규칙 지정
→ "이 포트로 오면 저 장치로 보내"
→ 외부에서 내부 장치 접근 가능 O
```

## 동작 방식

```
[설정]
공인 IP:8080 → 192.168.0.5:8080 으로 전달

[외부 요청]
외부 사용자 → 203.0.113.1:8080 접속
→ 공유기가 NAT 테이블 확인
→ 192.168.0.5:8080 으로 전달 O
```

## 실사용 예시

| 용도    | 외부 포트 | 내부 IP:포트           |
| ----- | ----- | ------------------ |
| 웹 서버  | 80    | 192.168.0.5:80     |
| HTTPS | 443   | 192.168.0.5:443    |
| 게임 서버 | 25565 | 192.168.0.10:25565 |
| SSH   | 22    | 192.168.0.5:22     |

## 포트 변환도 가능

```
외부 포트와 내부 포트를 다르게 설정 가능

203.0.113.1:9000 → 192.168.0.5:8080
→ 외부는 9000포트로 접속
→ 내부 서버는 8080포트로 수신
```

## 보안 주의사항

```
포트 포워딩 = 외부에서 내부로 구멍을 뚫는 것
→ 불필요한 포트는 열지 않기
→ 사용 후 설정 제거
→ 내부 서버 보안 강화 필수
```

> 포트 포워딩은 집에서 **마인크래프트 서버, 웹 서버** 등을 외부에 공개할 때 흔히 사용하며, 공유기 관리자 페이지(192.168.0.1)에서 설정할 수 있습니다.
