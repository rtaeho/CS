사설 IP 주소와 공인 IP 주소를 **1:1로 고정 매핑**하는 NAT 방식입니다.

## 동작 방식

```
[사설망]              [공인 IP]
192.168.0.1  ←→  203.0.113.1  (고정)
192.168.0.2  ←→  203.0.113.2  (고정)
192.168.0.3  ←→  203.0.113.3  (고정)

→ 항상 같은 공인 IP로 변환
→ 외부에서도 항상 같은 공인 IP로 접근 가능
```

## Static vs Dynamic vs PAT

||Static NAT|Dynamic NAT|PAT|
|---|---|---|---|
|**매핑**|1:1 고정|1:1 동적|다:1|
|**공인 IP 수**|장치 수만큼 필요|풀(Pool) 필요|1개로 가능|
|**외부 접근**|✅ 가능|❌ 어려움|❌ 어려움|
|**용도**|서버 운영|기업 환경|가정용 공유기|

## 언제 사용하나?

```
웹 서버, DB 서버처럼
외부에서 항상 같은 주소로 접근해야 하는 경우

ex) 회사 웹 서버
사설 IP: 192.168.0.10
공인 IP: 203.0.113.10 (고정)
→ 외부에서 항상 203.0.113.10으로 접근 ✅
```

## 단점

```
장치마다 공인 IP가 1개씩 필요
→ 공인 IP 절약 효과 없음
→ IP 부족 문제 해결에는 부적합
→ 서버처럼 고정 주소가 필요한 경우에만 사용
```

> Static NAT는 IP 절약보다 **외부에서 내부 서버에 고정된 주소로 접근**해야 할 때 사용하며, 일반 가정용보다 **서버 운영 환경**에서 주로 활용됩니다.
