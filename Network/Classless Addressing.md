IP 주소를 클래스(A/B/C)로 고정 구분하지 않고, **네트워크 크기를 필요에 따라 자유롭게 나누는 주소 체계**입니다.

## 등장 배경

```
클래스풀의 문제
→ 네트워크 비트 수가 클래스별로 고정
→ 필요한 호스트 수에 딱 맞는 클래스가 없음
→ IP 주소 낭비 심각

IPv4 주소 고갈 위기
→ 더 효율적인 주소 체계 필요
→ Classless 주소 체계 등장
```

## [[Classful Addressing]] vs Classless Addressing

||클래스풀|클래스리스|
|---|---|---|
|**네트워크 비트**|클래스별 고정 (8/16/24)|자유롭게 지정|
|**클래스 구분**|A/B/C 고정|없음|
|**IP 효율성**|낭비 심함|효율적|
|**유연성**|낮음|높음|

## 핵심 개념

```
클래스풀: IP 주소만 보면 네트워크 비트 수가 자동 결정
          10.0.0.1 → A클래스 → 네트워크 8bit 고정

클래스리스: 서브넷 마스크로 네트워크 비트 수를 별도 지정
           10.0.0.1 + 255.255.255.0 → 네트워크 24bit
           10.0.0.1 + 255.255.0.0   → 네트워크 16bit
           → 같은 IP라도 마스크에 따라 다르게 해석
```

## IP 낭비 해결

```
호스트 1000개 필요

[클래스풀]
C클래스: 254개 → 부족
B클래스: 65534개 → 64534개 낭비

[클래스리스]
서브넷 마스크로 딱 1022개짜리 네트워크 생성 O
→ 낭비 최소화
```

## 클래스리스를 가능하게 하는 기술

| 기술               | 설명                   |
| ---------------- | -------------------- |
| **서브넷 마스크**      | 네트워크/호스트 비트 구분       |
| **[[CIDR]] 표기법** | 서브넷 마스크를 /숫자로 간단히 표현 |
| **VLSM**         | 네트워크를 다양한 크기로 분할     |

> 클래스리스 주소 체계는 현재 인터넷의 표준으로, **서브넷 마스크를 통해 네트워크 크기를 자유롭게 지정**할 수 있어 IPv4 주소 고갈 문제를 완화했습니다.
