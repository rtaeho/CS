하나의 공인 IP에 **포트 번호로 여러 사설 IP를 구분**하여 다수의 장치가 하나의 공인 IP를 공유하는 NAT 방식입니다.

## 동작 방식

```
[사설망]                  [NAT 테이블]           [인터넷]
192.168.0.1:1234  →  203.0.113.1:5001  →  서버
192.168.0.2:1234  →  203.0.113.1:5002  →  서버
192.168.0.3:1234  →  203.0.113.1:5003  →  서버
        ↑                    ↑
   사설 IP:포트         공인 IP:포트 (포트로 구분)
```

## NAT 테이블 상세

```
출발지(사설)         변환(공인)          목적지
192.168.0.1:1234 → 203.0.113.1:5001 → 8.8.8.8:80
192.168.0.2:3000 → 203.0.113.1:5002 → 8.8.8.8:80
192.168.0.3:2000 → 203.0.113.1:5003 → 8.8.8.8:443

응답 수신 시
203.0.113.1:5001 → 192.168.0.1:1234 로 전달
203.0.113.1:5002 → 192.168.0.2:3000 으로 전달
```

## Static, Dynamic과 비교

||Static|Dynamic|PAT|
|---|---|---|---|
|**매핑**|1:1 고정|1:1 동적|다:1|
|**공인 IP 수**|장치 수만큼|풀 필요|1개로 가능|
|**구분 방법**|IP|IP|IP + 포트|
|**동시 접속**|항상|풀 소진 시 불가|거의 무제한|
|**용도**|서버|기업|가정용 공유기|

## 포트 번호 범위

```
포트 번호: 0 ~ 65535
→ 이론상 하나의 공인 IP로 약 65000개 장치 수용 가능
```

## 가정용 공유기가 PAT를 쓰는 이유

```
공인 IP 1개 (통신사에서 할당)
→ 스마트폰, 노트북, TV, 태블릿 등 여러 장치가 공유
→ 포트로 구분하여 각 장치에 응답 전달
→ 공인 IP 1개로 충분 O
```

## 외부에서 접근이 어려운 이유

```
외부 → 203.0.113.1 로 접근
→ NAT 테이블에 매핑 없음
→ 어느 사설 IP로 전달할지 모름
→ 포트 포워딩으로 해결 가능
```

> PAT는 **가장 널리 사용되는 NAT 방식**으로, 우리가 사용하는 공유기가 바로 PAT를 사용합니다. IP 주소 절약 효과가 가장 크기 때문에 IPv4 주소 부족 문제 완화에 핵심적인 역할을 합니다.
