클라이언트 측에 위치하여 클라이언트를 대신해 외부 서버에 요청을 보내주는 프록시 서버입니다.

## 동작 흐름

```
클라이언트 → 포워드 프록시 → 인터넷 → 목적지 서버
                ↑
    클라이언트의 IP를 숨기고
    프록시의 IP로 요청을 전달
```

1. 클라이언트가 프록시 서버에 요청을 보냄
2. 프록시가 목적지 서버에 대신 요청을 전달
3. 목적지 서버는 프록시의 IP만 인식 (실제 클라이언트를 모름)
4. 응답을 프록시가 받아 클라이언트에게 전달

## 주요 용도

|용도|설명|
|---|---|
|**IP 우회/익명성**|클라이언트의 실제 IP를 숨겨 익명으로 접속|
|**접근 제어**|회사/학교에서 특정 사이트 차단 또는 허용|
|**캐싱**|자주 요청되는 콘텐츠를 프록시에 저장하여 속도 향상|
|**로깅/모니터링**|사용자들의 인터넷 사용 기록을 추적|

## 대표 예시

- **회사 내부망**: 사내 프록시를 통해서만 외부 인터넷 접근 가능, 특정 사이트 차단
- **VPN**: 사용자의 트래픽을 VPN 서버(프록시)를 통해 우회
- **Squid Proxy**: 대표적인 오픈소스 포워드 프록시 소프트웨어

## [[리버스 프록시]]와의 핵심 차이

```
포워드 프록시: 클라이언트가 "누구인지" 서버에게 숨김  → 클라이언트 보호
리버스 프록시: 서버가 "누구인지" 클라이언트에게 숨김  → 서버 보호
```
