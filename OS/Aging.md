대기 시간이 길어질수록 **프로세스의 우선순위를 점진적으로 높여** 기아([[Starvation]])를 방지하는 기법입니다.

## 왜 필요한가?

```
우선순위 스케줄링 or SJF에서
높은 우선순위 / 짧은 작업이 계속 들어오면
→ 낮은 우선순위 / 긴 작업은 영원히 실행 못함
→ 에이징으로 해결
```

## 동작 방식

```
P1 (우선순위 10, 대기 중)

대기 10분 → 우선순위 9
대기 20분 → 우선순위 8
대기 30분 → 우선순위 7
...
대기 90분 → 우선순위 1 (최상위)
→ 반드시 실행됨 O
```

## 핵심 특징

|항목|내용|
|---|---|
|**목적**|기아(Starvation) 방지|
|**방식**|대기 시간 비례하여 우선순위 상승|
|**보장**|모든 프로세스가 언젠가는 실행됨|

## 적용 알고리즘

|알고리즘|기아 원인|에이징 효과|
|---|---|---|
|**우선순위 스케줄링**|낮은 우선순위 밀림|대기할수록 우선순위 상승|
|**SJF / SRTF**|긴 작업 밀림|대기할수록 우선순위 상승|

## HRN과의 관계

```
HRN 우선순위 = (대기 시간 + 실행 시간) / 실행 시간

대기 시간이 길어질수록 → 우선순위 자동 상승
→ HRN은 에이징 개념을 공식에 내포
```

> 에이징은 **공평성(Fairness)** 을 보장하는 가장 보편적인 방법으로, MLFQ 등 현대 스케줄링 알고리즘에도 폭넓게 적용됩니다.
