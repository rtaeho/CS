데이터를 암호화/복호화할 때 사용하는 키의 방식으로, 대칭키는 같은 키를 사용하고 공개키(비대칭키)는 공개키와 개인키 한 쌍을 사용합니다.

## [[대칭키]] (Symmetric Key)

```
암호화: 평문 + 키A → 암호문
복호화: 암호문 + 키A → 평문
→ 암호화/복호화에 동일한 키 사용
```

|항목|내용|
|---|---|
|**장점**|처리 속도 빠름|
|**단점**|키 전달 과정에서 탈취 위험 (키 배송 문제)|
|**대표 알고리즘**|AES, DES|

## [[공개키]] (Public Key / 비대칭키)

```
암호화: 평문 + 공개키(Public Key) → 암호문
복호화: 암호문 + 개인키(Private Key) → 평문
→ 공개키는 누구나 알 수 있고, 개인키는 본인만 보관
```

|항목|내용|
|---|---|
|**장점**|키 전달 문제 없음, 보안성 높음|
|**단점**|처리 속도 느림|
|**대표 알고리즘**|RSA, ECC|

## 대칭키 vs 공개키

||대칭키|공개키|
|---|---|---|
|**키 개수**|1개|2개 (공개키 + 개인키)|
|**속도**|빠름|느림|
|**키 배송 문제**|있음|없음|
|**주 용도**|대용량 데이터 암호화|키 교환, 인증|

## 실제 활용 ([[HTTPS]])

```
1. 공개키로 대칭키를 안전하게 교환
2. 이후 실제 데이터는 대칭키로 암호화 통신
→ 두 방식을 혼합해 속도 + 보안 모두 확보
```

> 공개키 방식은 **[[전자서명]]**에도 활용됩니다. 개인키로 서명 → 공개키로 검증하여 송신자를 인증합니다.
